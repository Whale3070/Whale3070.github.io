总结：

本次突破防线，利用的是后台提示用户名和密码太”贴心”。

如果提示”登陆失败，错误的用户名或密码。”

再加上一个强密码。（本次爆破成功，密码强度还可以，不算弱密码）爆破的成功率可以忽略不计。

学习了wpscan、nikto扫描。

不要依赖单一的工具。。

学习了一种新的获取webshell的思路

前景提要：上次通过爆破后台，修改404.php获得一个webshell。现在学习提权。

id、whoami、hostname

cd home/robot

MD5解码：abcdefghijklmnopqrstuvwxyz

(⊙﹏⊙)这个明显是偏CTF，而不是模拟真实渗透环境啊。
